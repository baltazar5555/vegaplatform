Universal rulesUniverzalna pravila

P1
Unique identificationEnotna identifikacija
Every document receives a unique ID. The ID never changes or gets recycled. The prefix determines the document type.Vsak dokument dobi unikaten ID. ID se nikoli ne spremeni ali reciklira. Prefix doloca tip dokumenta.
P2
Single Source of Truth
One central object for all counters. Modules never hardcode — they always read from the central source.En centralni objekt za vse stevce. Moduli nikoli ne hardkodirajo — vedno berejo iz centralnega vira.
P3
Automatic synchronizationAvtomatska sinhronizacija
When a new document is added, all modules update automatically. A human must not be the bottleneck for sync.Ob vnosu novega dokumenta se vsi moduli posodobijo avtomatsko. Clovek ne sme biti bottleneck za sync.
P4
Manifest tracking
The manifest records all shared data, who uses it, and current values. Drift = automatic correction.Manifest belezi vse deljene podatke, kdo jih uporablja in trenutne vrednosti. Drift = avtomatski popravek.
P5
Gate = structural onlyGate = samo strukturno
Review is for structural changes. Content entries without structural changes do not require review.Review je za strukturne spremembe. Vsebinski vnos brez strukturne spremembe ne rabi review-a.
P6
IdempotencyIdempotentnost
The same document twice through the pipeline = the same result. Duplicates are detected and rejected at step 3.Isti dokument dvakrat skozi pipeline = isti rezultat. Duplikati se zaznajo in zavrnejo v koraku 3.
